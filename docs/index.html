VoiceNative sits in your menu bar. It captures audio, transcribes locally on Apple's Neural Engine, and pastes the result into whatever app you're in. Audio never leaves your Mac.

Under the Hood

What we learned
building this.

We chose the Neural Engine.

Two paths exist on macOS: whisper.cpp runs on Metal (GPU), loads instantly. WhisperKit uses CoreML on the Neural Engine — a dedicated inference accelerator that doesn't compete for GPU or CPU time. For a menu bar app transcribing in the background, that matters.

The trade-off: CoreML compiles the model for your specific hardware on first load. 60–90 seconds, once. After that, ~5 seconds from cache.

First load: 97 seconds. We got it to 5.

The difference between broken and fast was a single directory path. Once we identified the correct cache location (~/Documents/huggingface/models/argmaxinc/whisperkit-coreml), subsequent loads dropped from 97s to ~5s.

The cache breaks if you switch compute units.

Switching from GPU encoder to ANE encoder invalidates the CoreML cache entirely. Another 90-second recompilation. We tried a hybrid — GPU for encoding, ANE for decoding. CoreML recompiled the full graph. We committed to full ANE for everything.

The configuration that stuck melCompute: .cpuAndGPU
audioEncoderCompute: .cpuAndNeuralEngine
textDecoderCompute: .cpuAndNeuralEngine
prefillCompute: .cpuOnly Mel on GPU (no ANE compilation needed). Encoder + decoder on ANE (fast sustained inference). Prefill on CPU (avoids ANE contention).

The model forgets you after 2 minutes.

macOS silently unloads CoreML models from the Neural Engine after inactivity. Next transcription: 30+ seconds. The fix: transcribe a 0.5-second silent buffer every 2 minutes. Invisible to the user. Keeps the pipeline warm.

/// Keep CoreML/ANE pipeline warm
func prewarm() async {
    let silence = [Float](repeating: 0, count: 8000)
    let options = DecodingOptions(
        task: .transcribe, language: "en",
        withoutTimestamps: true, suppressBlank: true
    )
    _ = try? await whisperKit.transcribe(
        audioArray: silence, decodeOptions: options
    )
}
Streaming Pipeline

A 2-minute recording
finishes in seconds.

Whisper processes a maximum of 30 seconds per pass. Without streaming, a 2-minute recording means 36 seconds of waiting. With streaming, we transcribe chunks in the background while you speak. Only the tail needs processing when you stop.

Without streaming

Wait = ceil(duration / 30) × 30 × 0.3
2 min recording = 36s wait

With streaming

Wait = (duration mod 30) × 0.3
2 min recording = 0–9s wait

The actual transcription speed didn't change. We moved the work from "after you stop" to "while you're still talking."
The Details

Audio. Hotkeys. Hallucinations.

Audio pipeline

Capture at 48kHz native rate via AVAudioEngine. Resample to 16kHz mono on demand using AVAudioConverter. Peak-normalize with vDSP (Accelerate framework). Without normalization, quiet mics produce samples Whisper reads as silence — triggering hallucinations.

Hotkey detection

NSEvent global + local monitors, not CGEventTap. CGEventTap requires Input Monitoring permission and silently fails when the app's own windows have focus. Right Shift is a modifier (keycode 60), detected via .flagsChanged, not .keyDown.

Hallucination filter

Whisper hallucinates on silence — "thank you for watching", repeated n-grams, non-ASCII output. We filter post-transcription:

  • Known phrase matching
  • N-gram repetition (3+ repeats)
  • ASCII ratio validation
